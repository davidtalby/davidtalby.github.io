 Syllabus:

The course of the first semester will be a theoretical one, and will cover these issues:

  • Introduction: The role of the OS, the hardware connection
  • Processes: Internal Representation, Creation, IPC
  • Short-Term Scheduling
  • Queue Theory: Basic statistics behind schedulers
  • Inter-Process Synchronization: Critical Sections, Semaphores and others
  • Deadlocks: Prevention, Avoidence and Detection
  • Memory Management: Hyrarchy, Paging, Segmentation
  • Virtual Memory: The hardware connection
  • I/O Devices and Secondary Storage Management
  • File Systems
  • Distributed Systems: Motivation, OSI seven layers model
  • Distributed Algorithms: Snapshot, Timestamping, Mutual Exclusion, Deadlocks
  • Security and Protection

The second semester will concentrate on practice - system programming in Unix. The subjects:

  • The Unix kernel architecture.
  • File system: Regular files, symbolic links and directories. The associated system calls.
  • File system implementation: Partitions, i-nodes and disk blocks, runtime data structures.
  • Processes: Creation, execution and termination
  • Signals: Common types, handlers, blocking, alarms
  • Short-term scheduler implementation
  • Memory management: Swapping and swapper process, virtual memory and the page daemon
  • Allocating user memory: Allocation map, powers-of-2 and the buddy system
  • Inter-process communication: Pipes, Fifos, the select system call, memory mapped files
  • TCP/IP Sockets
  • Security: The superuser, authentication, encryption, the Kerberos system


 Bibliography:

The books for the first semester (theory):

  • Operating Systems: Internals and Design Principles, by William Stallings, 3rd edition
  • Operating System Concepts, by Silberschatz and Galvin
  • Many other introductory books about operating systems exist, and most of them cover almost the same set of subjects. So if you find a book that you like, do use it, and recommend it in the discussion group.

The books for the second semester (practice):

  • The Design of the Unix Operating System, by Maurice S. Bach
  • Advanced Programming in the Unix Environment, by Richard Stevens
  • Advanced Unix Programming, by Marc J. Rochkind
  • Many other books describing Unix and NT in detail exist; any good reference of Unix's system calls will be useful.

Material handed out in class:

  • Summary of Unix System Calls for Files and I/O, in HTML and Word formats.
  • Summary of Unix System Calls for Process handling, signals and pipes, in HTML and Word formats.
  • Summary of Unix System Calls for Inter-Process Communication, in HTML and Word formats.
  • An example of a simple client/server program based on sockets, in Word or as server.c and client.c.

 Guidelines and Policies

The theoretical course will have weekly exercises, worth 15% of the final grade, and a final test that will determine the remaining 85%. In order to pass the course, all except one of the exercises must be submitted.

The practical course will have bi-weekly exercises, worth 30% of the final grade, and a final test that will determine the remaining 70%. In order to pass the course, all exercises must be submitted.

Read carefully the exercise guidelines for the second semester.

You are encouraged to submit in pairs.
