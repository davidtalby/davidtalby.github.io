Components Track

While the late 80's show the rise of Object-Oriented programming, the last 90's represent the rise of Component oriented programming: Develop independent software components which are defined by well defined interfaces, and build applications by combing such reusable components. Components could be written and used by different programming languages, loaded dynamically from a program that used them, and sometimes loaded from the net.

 

Main Lectures of this Track

1. JavaBeans

given by Elad Debby and Ronnen Shem-Tov

The competing technology to COM is platform independent, Internet capable, and is much easier to code since Java saves the programmer much of dealing with memory management, multi-threading, dynamic loading, security and  others. This lecture focuses on the Java Beans framework: Java classes and reflection; what is a bean; how to add properties and events to beans; how to write a simple bean, and how to use it; how beans support persistence; the Java.beans package; and a short overview for more specialized features, such as property sheets, the PropertyVetoException mechanism, and writing BeanInfo classes.

The two demons of this lecture must be one that shows how to write a Java bean, and another that shows how that bean is used - properties and events included. Demonstrating other features will award bonuses.

2. ATL

given by Markov Avraham, Galit Tal and Merav

Create COM componets in Visual C++ 6.0 using the Active Template Library 3.0. Covers the purpose of ATL, the available wizards, the syntax of TLB files, the #import directive, other file types created in a project, adding properties and events, and a discussion of when C++ is preferrable to VB for developping COM components.

The demos of this lecture must include: a C++ ATL project including properties and events that is used by an InterDev project; and another ATL project that demonstrates a COM component that adds and removes support for interfaces at runtime using a non-trivial QueryInterface() function.

3. COM

given by: Eran Amzelag and Shlomi Armon

This lecture presents Microsoft's Component Object Model: The basic definitions of interfaces, properties; IUnknown, reference counting, and QueryInterface(); IDispatch; the technical standard for representing COM objects implementing multiple interfaces in memory; the registry; and what the Windows operating system includes to support COM.

The practical aspect of this lecture must include demos of these two abilities: a simple Visual Basic 6 application that demonstrates use of the Office Automation objects; and the creation, registration and use of a simple COM object in VB6 that implements one simple interface with methods, properties and at least one event.

3. DCOM

Given by Rotem Cohen, Nir Shani and Roi Cohen

Create and used Distributed COM: The ability to easily deploy COM objects over a network. Covers technically doing this via the registry; configuring security; inter-thread and inter-process synchronization; design principles; performance issues; and a discussion of when DCOM is useful and when it is not.

The two demons of this lecture are left to the lecturers' discretion. They should provide simple yet practical examples of using DCOM, and be very detailed so they could be used as a self study tutorial to other students. Online links:

Short Lectures of this Track

The short lectures of this track would be better named the "Platforms" track - they discuss how applications and components can be written for hand-held and cellular devices. These are very complex technologies to which we donate very little time, only because we don't have much time and because the number of users in each technology is still much smaller than those of the others we survey.

1. Dynamic Link Libraries (DLL)

given by Gadi Nehemne, Tami Jacoby and Nitza Maymon

A short explanation of what is a DLL (that is not necessarily a COM component). Also explain what the Registry is, and how it works.

2. Java Archives (JAR Files)

given by: Yonatan Zadik, Michael Saban and Dror Cohen

What are they, how they are used, how they are dynamically loaded, cryptographically signed and used.

3. Java Naming and Directory Service (JNDI)

given by Itamar Gus

This service is the Java counterpart of the Windows registry - it allows registering and activating components (Java beans) on remote computers. Explain what it is, when it should be used, and how it is programmed (i.e. how to register and how to lookup components).

4. Active Directory

given by: Eran Abramovitz, Gary Rosner & Lidar Reuvenof

Windows 2000 introduces the Active Directory as a central storage of the administrative settings of a distributed network. Explain what the active directory is, what is stored in it, and in what ways this improves over earlier systems.

 

Projects of this Track

A project in the components track should demonstrate a combination of several abilities of components, such as language independence, the ability to work over a LAN, and dynamic loading. For example, one might offer a project that includes a COM server object which does something useful (schedules backups, prints Office documents, sends Outlook email, etc.), and two client applications to access it, one written in VB and another with InterDev. Or, one can offer a JavaBeans project for some database server, for which new abilities can be added at runtime by loading new beans that implement familiar interfaces, and a client that can query and then use such abilities. Projects that replace some of the above with dynamically changing security policies, or that offer a PalmPilot companion to a desktop application, would also be welcome.

In short, projects must be non-trivial, demonstrate more than one of the advantages of components, and be a useful application (although it can be a very simple one).
