Active Web Track

In 1995, Netscape 2.0 became popular, and it was possible to pages with text and pictures from all over the world using your home phone. The Internet boomed. Four years later, the web is an interactive experience: You can show, chat, vote, personalize, get live video news, and run complex queries and searches. Intranets also "happened" and now everyone has one; they enable scheduling appointments, exchanging ideas, searching corporate databases and online customer service. This track presents the technologies that make the web active, enabling complex two-way communications between people and web servers.

 

Main Lectures of this Track

1. Scripting

given by Ana Ternovski and Yifat Ezra

This lecture presents both the JavaScript world (components, examples, servers), and the JScript/VBScript (Microsoft) world of web page scripting. Starting with a discussion of when scripting is appropriate and how it is best used, the lecture continues with giving examples from both worlds, and then by comparing the languages, development tools, browser support, servers and industry use.

The two demos should be one in JavaScript and another in VBScript. They should both include non-trivial graphical web pages, and controls that receive user events and answer them somehow.

2. ASP / JSP

ASP given by Itamar Gus

JSP given by Arik Peled and Guy Amir

What is it, which applications is it best for, how it works, which development tools and languages are supported, how does the server work, and a glimpse into ASP+. The lecture should include full examples.

The two demos should demonstrate two different features of ASP. At least one should involve accessing a database (via ADO).

given by Sara Sigel, Haviv Rubin and Esty Gants

How to build one, how it interacts with COM, security, deployment, properties & events, programming environment

4. DHTML

given by Gadi Nachmana, Nitza Maymon and Tami Ya'akovi

The Dynamic HTML was the bridge that allowed writing web pages that ran over both IE and Netscape, in their 4.0 version. It allowed several visual controls, forms, and some graphics/motion capabilities. All this didn't require any (other) programming, and several tools allowed easy creation and editing of such sites. This lecture covers all this, including examples and an overview of leading development tools.

Two examples of DHTML. Combination with XML recommended.

Short Lectures of this Track

The short lectures of this track discuss evolving new directions in the Active Web track - XML and XSL based standards, as well as other ways for people to communicate over the Internet that are not based on working through a web browser.

1. Cascading Style Sheets (CSS)

given by Yitzik Csillag and Roy Ben-Yeheskel

What are they, what they were meant to do in addition to HTML, DHTML and scripting languages, and which development environments support them.

2. Java Applets

given by: Galit Tal, Merav Tzviri and Avrham Markov

An introduction and an example, followed by a technical comparison with ActiveX in terms of security, platform independence, ease of development, performance, browser support, and so on.

3. Java Servlets

given by: Arik Peled & Guy Amir

What servlets are, how they are programmed in Java (with code samples), and which popular servers exist to run them.

4. Napster and Gnutella

given by: Mickey Raviv and Guy halfon

The advantages of peer-to-peer direct communications; the algorithms and administration behind it; the IBM and Microsoft effort to offer such infrastructures.

 

Projects of this Track

A project in this track would naturally comprise of a web site that includes personalization, chat, file exchange, shopping or another active web activity. It should include a good generic design, and be easy to install, administrate, and expand. Projects in this track should demonstrate more than one technology: a comparison of scripting and XSL over DHTML, a peer-to-peer implementation working from within an applet, a meeting organizer via exchange and a web client that enables remote access to your appointments, or other proposals of similar scope.
