Security Track

Ten years ago, issues such as public key exchange, trusted third parties, code exchange between untrusting parties, impersonation, electronic cash, dynamic signatures and zero-knowledge proofs were mostly of academic interest. And then was the Internet.

 

Main Lectures of this Track

1. Java Security Architecture

given by Asaf Eisner, Yaron Zalikovits and Orit Mouki

The Java security framework and security architecture - policies and their supported types, SecurityManagers, the java.security.* packages, applets, and examples of code and tools that allow editing fine-grained policies.

Demos: one applet/application that demonstrates different fine-grained security violations, and one self-written security manager or policy file.

2. Java Cryptography Architecture

given by Orli Rosentsvieg and Liran Nir-Shoshan

The cryptography architecture of Java, code signing (for JARs and applets), description of the available classes and interfaces (in java.security.*), available implementations, code examples, and available tools.

The demos: One of authentication and digital signatures between two applications, and another one of signed classes or applets (possibly using jarsigner).

3. WindowsNT/2000 Security Model 

given by Oren Gordon and Arik Shitrit

How Windows 2000 handles authentication, autorization, logon, remote access, smart cards, policy management, auditing, interoperability with other operating systems, and a secure file system (NTFS security). Explain how programmers and administrators can use tokens, the active directory, impersonalization levels, security templates and other mechanisms to administer a secure network.

The demos can be coded, to show remote access or interoperability, but can also show how to use existing tools to manage security properties. In any cases, the two demos must each show different features of the security model.

Windows 2000 Security Related Articles

4. Virus Programming

given by Ana Rivechkov, Chani Kaspi and Hadas Harel

Categories of existing viruses, how they are developed, and how they are distributed over the web. Code examples of recent viruses and disguising techniques. Libraries for writing viruses.

Demos: Code samples of existing viruses, demonstrating different distribution techniques. Preferably one of them must use a virus programming library that is available online.

Short Lectures of this Track

While the major lectures in this track cover material most useful to programmers and designers, the short lectures add material about products that try to minimize and even prevent programming. Some require small programmer involvement, but most are purely a system administrator's job.

1. Secure Socket Layer (SSL)

given by Ariel Vine and Noam Gerber

What is it, when is it appropriate, how authentication and encryption work, how to write a web site that has it.

2. Kerberos

given by: Anna Tarnavskaya and Yifat

What is it, how it works, which systems use it, what developers need to know to work with it.

3. Firewalls

given by Sagiv Freimark, Shiran Levy and Ziv Balshai

What is it, when to use it, how it's implemented, what it prevents and what it doesn't.

4. Virtual Private Networks (VPN)

given by Eduardo Vainroj, Shimrit David and Ayellet Elbaz

What is it, when to use it and when not to, main features of popular products, and expected future improvements.

 

Projects of this Track

Projects in the security track should combine at least two of the surveyed technologies and tools to create a secure web or Intranet application. An online bank, which requires authentication, encryption and web server protection together is an interesting project; other offers of similar scope are invited.
