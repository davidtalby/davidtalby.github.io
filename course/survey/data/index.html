Data Exchange Track

Having a standard, widely available communication channel between computers in your business (LAN, or an Intranet) and in the world (the Internet) enabled data exchange in many formats and standards; for example, web pages usually use HTML and email uses POP3. This track presents other widely used data exchange standards, all in wide use by enterprise applications of all kinds.     

 

Main Lectures of this Track

1. XML and XSL

given by Gary Rosner, Lider Reuvenof and Eran Abramovitz

XML; schemas (and a short mention of DTDs); validation; parsers; XSL samples and the XML-QL query languages; XSL templates; using XML and XSL to build efficient active web pages.

Demos: validation, intersections and updates by keeping objects as XML in an app; using XSL to produce DHTML pages.

Bibliography:

Goggle search for XML (all are relevant...)

2. ADO

given by Idit Sharaby, Itzik Shpizzen and Nissim Cohen

What; how; recordsets & commands; queries; other database activities; technical implementation; handling of conflicts in disconnected recordsets; improvements in ADO+; comparison to JDO.

Two demos, one using connected recordsets and another one using disconnected recordsets.

Bibliography:

Universal Data Access Web Site 

ADO Tutorial

3. PalmPilot

given by Ariel Vine, Noam Gerber and Eliram Benitah

How to develop applications for the PalmOS, including debugging, installation, and synchronization with a PC.

The demos must include the full source code of a simple Palm application, and detailed instructions about installing and extending it. Second demo should include synchronization with the Internet (through a PC).

Bibliography:

An Introduction to Programming the PalmPilot (all the important URLs are at the end of the article!)

CodeWarrior for the Palm OS (the lite version is a free download)

4. Exchange

given by: Yitzik Csillag and Roi Ben-Yehezkel

Developing applications over Microsoft's Exchange Servers; Outlook customizability. Which applications are appropriate for this easy-to-develop approach?

Two demos: One inside Outlook, one independent but still over Exchange.

Bibliography:

MS Exchange Server Online Developers Center

Exchange Server Technical Articles

Short Lectures of this Track

The short lectures of this track would be better named the "Platforms" track - they discuss how applications and components can be written for hand-held and cellular devices. These are very complex technologies to which we donate very little time, only because we don't have much time and because the number of users in each technology is still much smaller than those of the others we survey.

1. Microsoft Message Queues (MSMQ)

given by Vladimir Green and Marina Chernomez

This API and service is built into Windows 2000 and the .NET platform and allows components, on the same or remote computers, to send each other asynchronous messages. This topics explains how to do this, and for which applications it is useful.

2. Java Data Objects (JDO)

given by: Sarra Knobloch, Haviv Rubin and Esty Gants

Also given by: Itzik Shpizen, Nissim Cohen and Idit Sharabi

This is the Java alternative to Microsoft's ADO; explain how it works, and compare it with its competitor. 

3. Windows CE

given by: Moti Raz, Shachar Erez and Tomer Yatskan

An overview of the Windows CE operating system for hand-held devices, compared with the PalmOS. Includes a short overview of what CE includes in comparison with desktop Windows, and design principles for applications that must support both desktop and CE user interfaces.

4. JINI

given by: Yuval Kadosh and Guy Jarden

What is JINI, what can you actually develop with it today, what are the future plans, and how BlueTooth enters the picture. This is mostly an industry research of future directions, so any demo of practical use would be welcome. 

 

Projects of this Track

A project in the data exchange track should combine at least two of the above subjects, or related ones that you can offer, into an interesting application. For example, you can write a server which publishes data in XML, and two clients that use it, one inside a web browser using XSL and another inside Outlook that dynamically builds a table and displays the data. Another example would be an application that synchronizes between PalmPilot or a CE machine and an exchange or a JINI server. An acceptable ADO project would be to use ADO+ which automatically sends XML in a three-tier application without programming.
