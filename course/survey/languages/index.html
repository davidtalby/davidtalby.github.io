Languages Track

A decade ago, developers would choose a programming language, write their application's entire source code, compile it, link it to create an executable on some machine and operating system, and run it. Today, not one part of this sentence remains fully correct - code is heavily reused in different programming languages and platforms, parts of the program are loaded from the web at runtime, programs can (and do!) fix themselves while running, and linking is often dynamic as well. This track covers these drastic changes in the way programmers work.

 

Main Lectures of this Track

1. CLR and CSharp

given by Yaron Basson, Yosi Cohen and Nariman Cohen

Microsoft's Common Language Runtime in the .NET platform is an ambitious attempt to build a tower of babylon of all programming languages. Over 30 languages are expected to be supported. The major development languages for the Windows and .NET platforms are also new - Visual Basic 7 is very different than its predecesor, and C# (pronounced C-Sharp) is an all new language. This lectures covers the main features of the CLR (classes, inheritance, exceptions, threading, garbage collection, shared libraries, etc.), and an overview of the main features of the C# programming languages.

One of the demos should be consist of two classes written in C# which interact using events and properties. The other should show how unmanaged and managed C++ code in Visual Studio.NET can be combined in a single executable.

Bibliography:

http://msdn.microsoft.com/net/

2. Delphi & Kylix

given by Uzi Chen, Eliaz Hershkovits and Itamar Cohen

Delphi is a very powerful RAD development environment for Windows, which improves Visual Basic by providing a better programming language (based on Object Pascal), and making it much easier to write new user interface components. Over the last few years, a very large number of smart user interface components have been written for Delphi, making it today the prime choice for creating fancy UIs.

Today, Delphi has been ported to Linux in a project called Kylix. This creates the most advanced development environment on Linux today, making it far easier to develop fast applications with complex user interfaces, and to develop Apache-based Internet and database applications. Moreover, libraries that can be shared between Windows and Linux have been developed, enabling developers to write code once and compile it to both platforms.

This lecture should cover Delphi's programming language (classes, inheritance, interfaces, exceptions, etc.), the ability to write components (how it is coded and used), the CLX library that is shared between Windows and Linux, and design tips for writing applications that can run on both platforms.

Of the two demos, one should be a Windows application demonstrating the use of some freeware Delphi components (from the list below), and another should either be in Linux (if possible) or demonstrate Delphi's polymorphic features.

3. Java 1.0 and 1.1

given by: Barak Hecht and Shay Amir

The Java language brought into mainstream programming many features and additions that were considered academic or experimental until them. It had seem tremendous success thanks to these innovations. This lecture covers the major features that helped Java gain its initial popularity: The Java Virtual Machine and Bytecode, the ability to create both applications and applets, garbage collection, dynamic class loading (and class ClassLoaded), Reflection, Serialization, the configurable Security Manager, and the java.net.* class library.

One of the demos of this lecture should show a practical use of reflection and serialization (and XML object stream, for example), and the other should be an applet that can present data about a given URL (such as its title and first few lines).

Everything you need is under "Products & APIs" in Sun's Java site: 

java.sun.com

4. Java 1.2 and 1.3

given by: Available

Java continued to implement very new ideas in programming languages, and in contrast to many other such attempts, they are widely used all over the world. This lecture covers the major additions in the 1.2 and 1.3 revisions of the JDK, relying on the standard edition of the platform (J2SE). The topics covered are: Java Beans, the Collections Framework, dynamic proxy classes, reference objects, Swing, the accessibility API, the extensions API and the plug-in framework.

One of the demos of this lecture should demonstrate a practical use of dynamic proxy classes, and the other should show a practical use of reference objects.

Online links (like all other subjects, this should be expanded after the lecture):

java.sun.com/j2se/1.3/docs

Projects of this Track

A project in the languages track requires you to learn and employ some of the more sophisticated features of the discussed languages, in a way that creates a useful application (not just a useless demo of some weird features). Projects that combine two programming languages are most welcome. For example, a project can demonstrate how MIL code allows writing a GUI in VB, an algorithm in C++ and a data structure in CSharp. Another project can demonstrate how Java objects can be serialized to XML using Reflection and communicated to and from Delphi code via sockets. Or, as usual, you can make your own offer.
