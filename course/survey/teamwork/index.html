Teamwork Track

Software today must be more complex, more distributed, more flexible, have a very short time-to-market, and face changing requirements more rapidly. More and more, it is written by large groups of people with very different abilities, sometimes in different locations. Yet, customers are less understanding than ever about quality and bugs. This track covers some of the tools and methods developed to make software teams more efficient and fun.

 

Main Lectures of this Track

1. Configuration Control

given by Ilan Pe'er and Noam Glan

Explains the problems of developing in large teams, and how ClearCase, and the UCM model in particular, help in solving these problems. Demonstrates how these solutions work in projects of different sizes and types. The lecture should also mention ClearQuest and ClearCase MultiSite.

The demos should be two case studies of two different real-world large projects, their use of configuration control, and the related problems that were encountered.

Rational ClearCase Homepage

Eric J. Ostrander's ClearCase/ClearQuest Page

2. Automated Testing

given by Amir Levi, Michel Moshayev and Kobi Levi

An overview of the test requirements of large projects: Unit tests, integration tests and stress tests. Shows how to use Mercury's tools to plan, write and manage both manual and automatic tests. Learn LoadRunner, WinRunner, ActiveTest, TestDirector and Astra (which is available for download).

Demos should include samples automatic test scripts and stress test examples.

MercuryInteractive.com

3. Rational Unified Process

given by: Available

The RUP is a software development methodology that is also a product. This lecture explains what the RUP includes, how it should be used, which projects can benefit most from it, and how Rational's products interact with it.

The demos should be two case studies of two different real-world large projects, the problems they encountered, and a critical analysis of whether RUP could have helped.

Rational RUP Homepage
What is the RUP?
Completing the Unified Process with Process Patterns

4. Extreme Programming

given by Ben Dvir, Dor Sher and Alon Yakutiel

XP is a new software development methodology that emphasizes very simple, fast, well-tested and ego-less teamwork. This lecture presents the principles of XP, success stories so far, and tools that support this way of work.

Of the two demos, one should be a written report of a real project that used XP successfully, and another should be a small application that uses either JUnit or a simpler framework for integrated unit testing.

Short Lectures of this Track

The short lectures focus on areas which are just as important as the major ones, but are not as commonly used.

1. Defect Management

given by: Itamar Cohen, Uzi Chen and Eliad Hershkovits

Compare Rational ClearQuest and Mercury TestDirector as tools to manage the work of software teams, track and report bug and tests.

2. Profilers

given by Oren Gordon and Arik Shitrit

What is it, how it is used, and when it should be used. Use Rational Quantify to demonstrate your points.

3. SourceSafe

given by: Eran Amzallag & Shlomi Armon

Microsoft's configuration control tool, bundled together with the Visual Studio. 

4. Static Analysis

given by: Barak Hecht and Shai Amir

Describe the leading tools for static analysis of source code for C, C++ and Java, and explain what static analysis tools are, and for what purposes they should be used.

 

Projects of this Track

Projects in this track would extend the discussed tools (or others) to extend their performance. This should be more than script-writing for a test suite - a better example of a project would be an application that manages manual and automatic test scripts and results in a configuration controlled project, or an application that manages the deployment of a large software to different customers based on versions and using shared components from a repository. Projects should interact with at least one commercial application in the area.
